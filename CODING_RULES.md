# Правила кодирования (CODING_RULES.md)

## Запрещённые действия

**Строго запрещено изменять, удалять или перезаписывать любые существующие файлы в проекте**, за исключением файлов документации Sphinx в папке `docs/`.

### Список директорий с существующими файлами, которые нельзя трогать:

- `/workspace/adf2stowf/` — все файлы (`adf2stowf.py`, `__init__.py`, `stowfn.py`, `adfread.py`)
- `/workspace/kfreader/` — все файлы (`kf.py`, архивы и др.)
- `/workspace/examples/` — все поддиректории и файлы внутри:
  - `Xe/`, `N/`, `H/`, `O3/`, `Be/`, `Ne/`, `Kr/`, `H2/`, `Ar/`, `He/`
  - Все файлы `.data`, `.svg`, `.asc`, `.txt`, `.out` и другие
- `/workspace/` — корневые файлы:
  - `README.md`
  - `pyproject.toml`

## Разрешённые действия

Разрешено **только**:
- Создание новых файлов документации Sphinx **исключительно в папке `/workspace/docs/`**
- Генерация конфигурации Sphinx (`conf.py`, `index.rst`, `.rst` файлы) внутри `docs/`
- Добавление новых файлов `.md` или `.rst` только внутри `docs/`

## Приоритет правил

1. Никогда не модифицировать существующие файлы вне `docs/`
2. Не создавать файлы кода (.py, .sh, и т.д.) вне `docs/`
3. Вся новая разработка кода запрещена — только документация Sphinx в `docs/`

## Примечание

Перед выполнением любых действий проверяй:
- Существует ли файл? → Если да, и он не в `docs/`, не трогай его.
- Находится ли целевая директория вне `docs/`? → Если да, не создавай там ничего.
