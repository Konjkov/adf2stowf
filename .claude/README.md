# .claude/ — конфигурация проекта для Claude

Эта папка содержит конфигурацию для работы с Claude.
Структура намеренно совместима с **Claude Code** (CLI) как основным инструментом.

---

## Структура

```
.claude/
├── README.md                        # этот файл
└── skills/
    └── adf2stowf/
        └── SKILL.md                 # доменная документация проекта
```

---

## Почему `.claude/skills/<name>/SKILL.md`, а не `.claude/SKILL.md`

### Нотации, которые существуют

| Нотация | Где работает | Как вызывается |
|---|---|---|
| `.claude/SKILL.md` | claude.ai веб-чат (системный маппинг `/mnt/skills/`) | автоматически, по триггерным словам |
| `.claude/commands/<name>.md` | Claude Code CLI | `/name` — устаревший формат |
| `.claude/skills/<name>/SKILL.md` | Claude Code CLI | `/name` — **текущий рекомендуемый формат** |

### Почему веб-чат и CLI используют разные нотации

**claude.ai** загружает скиллы через собственную серверную инфраструктуру: файлы
из репозитория синхронизируются в `/mnt/skills/user/` внутри сессии. Маппинг
происходит вне репозитория — сам файл может лежать где угодно.
Поэтому исторически достаточно было одного `.claude/SKILL.md`.

**Claude Code** (CLI, локальный процесс) читает файлы напрямую из файловой системы
проекта. Нотация с подпапкой `.claude/skills/<name>/` появилась позже (2025)
как архитектурное решение:

- подпапка позволяет хранить рядом со `SKILL.md` вспомогательные файлы
  (`examples/`, `scripts/`, `template.md`) — это полноценный «пакет» скилла
- `name` директории → имя slash-команды `/name`
- скилл может вызываться и явно (`/adf2stowf`), и автоматически Claude
  по триггерам из `description` во frontmatter

### Источник путаницы

Проект изначально создавался с `.claude/SKILL.md` (плоская нотация для
веб-чата). Claude Code пришёл позже и принёс другую структуру.
Обе нотации «работают» в своих контекстах, но выглядят несовместимо.

**Решение:** хранить в репозитории каноническую нотацию Claude Code
(`.claude/skills/<name>/SKILL.md`). Веб-чат синхронизирует содержимое
`SKILL.md` независимо от того, в какой подпапке он лежит.

## Использование в Claude Code

```bash
# явный вызов скилла
/adf2stowf

# или Claude подгрузит автоматически при упоминании:
# TAPE21, stowfn.data, CASINO, STO, cart2harm, cusp, zeta...
```
